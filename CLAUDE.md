## 3. Core Features (DO NOT BREAK)

| Feature | Where | Notes |
|---|---|---|
| Paste URL → auto-fetch (500 ms debounce) | `App._on_url_change` | YT_URL_RE matches watch/shorts/live/playlist/youtu.be |
| Format & quality picker (MP4 up to 4K, MP3 128/192/320 kbps) | options card | `_read_format_choice` |
| Unlimited parallel downloads | each `DownloadItem` spawns its own `threading.Thread` | Qt signals marshal to main thread |
| Cookies (none / cookies.txt / Firefox / Brave / Chrome / Edge) + Test login | `AccountDialog` | `App._cookie_opts` |
| MP3 with cover-art + metadata | `DownloadItem._run` | `writethumbnail` + `EmbedThumbnail` + `FFmpegMetadata` PPs |
| Clipboard watcher (QClipboard.dataChanged) | `App._on_clipboard_changed` | toggleable in Settings |
| Session persistence | `~/.yt_downloader_downloads.json` | `to_dict/from_dict`, mid-download → "interrupted" on restart |
| Conflict resolution (Replace / Rename / Cancel) | `conflict_dialog` | pre-checks `sanitize_filename(title).<ext>` |
| Duplicate detection (same `extract_video_id` + completed) | `duplicate_dialog` | Yes/No prompt |
| Thumbnails (cover-fit to 104×64 via PIL → QPixmap) | `_ThumbLoader` worker | duration badge overlay |
| Action menu (kebab) per row: Retry, Play, Show in folder, Copy link, Rename, Remove | `DownloadRow._show_kebab_menu` | state-driven |
| aria2c integration (16 parallel connections) | `DownloadItem._run` | falls back silently if not on PATH |
| Playlist support with checklist UI | `App._playlist_fetched`, `_add_playlist_downloads` | `extract_flat='in_playlist'` |
| Search / sort / filter tabs (All/Video/Audio) | filter row in `_build_ui` | `_refresh_list` |
| Pagination (15 / 30 / 50 / All per page) | `_render_pagination` | chevrons flip in RTL |
| System tray (close → hide, right-click menu, batch-completion notify) | `ui/tray.py` | `setQuitOnLastWindowClosed(False)` |
| Live i18n switch (no restart) | `App.set_language` → `retranslate` | every widget has a retranslate path |
| Auto-update (GitHub releases) | `updater.py` + title-bar dot + pill | silent on auto-check failure, loud on manual Check-for-updates |
| About dialog (credits, donate link, version) | `AboutDialog` | clickable URLs via `QDesktopServices` |

**Real file size after completion** — `DownloadItem._resolve_final_path()`
walks `info.requested_downloads` from `extract_info(download=True)`, then
falls back to expected filename, then a folder scan. NEVER falls back to
`size_bytes` from the progress hook (which is a single stream, not the merge).

**File-size scan strips both `\W` AND `_`** because yt-dlp commonly turns
spaces into underscores.

## 5. Window Chrome Architecture

### Frameless main window

`FramelessMainWindow(QMainWindow)` in `ui/window_chrome.py`:
- `Qt.Window | Qt.FramelessWindowHint`
- **No** `WA_TranslucentBackground` — we use a `setMask` with `QPainterPath.addRoundedRect(rect, 12, 12)` on every `resizeEvent` to clip to rounded corners.
- `nativeEvent(WM_NCHITTEST)` returns `HT*` on edges (6 px resize border) → native edge-resize + Aero snap, `HTCAPTION` over the title bar (except buttons) → native drag.

### Title bar (44 px) layout

LTR: `[controls] [brand_wrap] [drag_area, stretch=1] [right_wrap]`
RTL: `[controls] [right_wrap] [drag_area, stretch=1] [brand_wrap]`

`TitleBar.apply_layout_direction(direction)` re-orders the three middle
clusters. The `TitleBar` widget itself is locked LTR; only the cluster
order swaps.

**Window controls** — `WindowControls` container is `setLayoutDirection(Qt.LeftToRight)` **always**, so the traffic lights stay pinned to the visual left even in RTL.

| Button | Size | Color | Hover | Icon (mdi6) |
|---|---|---|---|---|
| Close | 32×24, r=6 | `#EB5757` | `#FF6B6B` | `close` 14 px, `rgba(0,0,0,0.6)` |
| Maximize | 32×24, r=6 | `#27A644` | `#34C759` | `fullscreen` (or `window-restore` when maximized) |
| Minimize | 32×24, r=6 | `#F0BF00` | `#FFD028` | `window-minimize` |

Order **close → max → min** left-to-right.

Both `setFixedSize` and QSS `min/max-width: 32` `min/max-height: 24` because Qt layouts can stretch.

**Brand cluster** — `"Nazzil"` (14 px `#828FFF` 500) + 10 px `●` status dot (gray default; updater flips green/yellow) with `QGraphicsDropShadowEffect(blur=8, alpha=102)` glow + optional `Update vX.Y.Z` pill.

**Right-side icon buttons** — 32×32 each, 17 px qta icons, compact / cookies / settings.

### URL row (60 px)

`[QLineEdit#UrlEntryV2 (leading mdi6.link)] [QPushButton#FetchBtnV2 = "Fetch"/"جلب"]`

Both `min-height: 40 px`. Fetch is brand-colored, `border-radius: 8 px`,

### Filter row (52 px)

`[All·N] [Video·N] [Audio·N] [<stretch>] [SearchEntry (140-280 px)] [Sort (120 fixed)] [Clear ✕ (red)]`

The search input is **always visible** — it's a real `QLineEdit` with leading magnifier qta icon and `textChanged → _on_search`. Earlier "expand-on-click" version was abandoned.

### Download row (comfortable 92 px / compact 68 px)

Three columns:

```
┌──────────────────────────────────────────────────────────┐
│ [thumb 104×64]  Title (elided right)         [📁] [⋯]    │
│  ⏱ 4:32         [YouTube] Ch · [MP4·1080p] 12 MB · 2 MB/s│
```

- Thumbnail `104×64` (comf) / `68×40` (compact), border-radius 8.
- Title is `_ElidingLabel` (`QFontMetrics.elidedText(... ElideRight)`),

- Action buttons 30×30 (QSS content 28 + 1 px border × 2), icon 16 px.
- Row min height enforced: `setMinimumHeight(92 / 68)`.
- Hover state: `[state="downloading"]` brand tint, `[state="failed"]` red border.

## 8. i18n Rules

- `i18n/en.json` and `ar.json`: **166 keys each, zero diff**.
  Verify with:
  ```python
  set(json.load(open("i18n/en.json"))) == set(json.load(open("i18n/ar.json")))
  ```
- `Translator.load(code)` reloads bundle; English is the fallback.
- `t(key, **kwargs)` does `str.format()` on the value.
- **Live switch** — `App.set_language(code)` calls `Translator.load`,
  `QApplication.setLayoutDirection`, `_apply_window_direction`, then
  `retranslate`. Every dialog has a `retranslate()` method including
  `set_title()` for the custom title-bar text. The guard
  `if code == Translator.lang(): return` short-circuits same-language
  calls — important when writing test code.
- **RTL** — `setLayoutDirection(Qt.RightToLeft)` on the window
  propagates to children. `WindowControls` is force-LTR so traffic
  lights stay on the left. `TitleBar.apply_layout_direction` re-orders
  the brand/right clusters. Pagination chevrons swap. Dialogs inherit.
- **Arabic alignment** — Use `Qt.AlignLeading | Qt.AlignVCenter` (NOT
  `AlignLeft`) so labels right-align in Arabic naturally.
- **Unicode-aware ellipsis** — `_ElidingLabel` uses
  `QFontMetrics(font).elidedText(text, Qt.ElideRight, width)` because
  manual `text[:n] + "…"` breaks on Arabic and emoji.

## 9. Build & Release Workflow

### Dev

```cmd
python main.py            # from inside nazzil/ — fast iteration
..\start.bat              # from anywhere via the outer wrapper
```

### Local build

```cmd
nazzil\build.bat          # → dist\Nazzil.exe (PyInstaller --onefile --windowed)
```

### Release (user runs this)

```cmd
release.bat               # bumps VERSION minor, commits, pushes, gh release create
```

Triggers `.github/workflows/build.yml` which:
1. Checkout
2. Setup Python 3.11 + install requirements + pyinstaller
3. `python generate_icon.py` (with `QT_QPA_PLATFORM=offscreen`)
4. `pyinstaller nazzil.spec` → `dist/Nazzil.exe`
5. `choco install innosetup -y --no-progress`
6. `ISCC.exe installer.iss` → `output/NazzilSetup.exe`
7. Upload BOTH assets to the GitHub release via `softprops/action-gh-release@v2`

### Auto-update flow (in-app)

- On startup, `App.start_update_check()` silently queries
  `https://api.github.com/repos/Hjbiki/Nazzil/releases/latest` (10 s timeout).
- On `update_available` → title-bar dot turns yellow, `Update vX.Y.Z` pill appears.
- User clicks pill → if frozen + asset_url, `UpdateDownloader` streams the
  exe to `%TEMP%\Nazzil_update.exe`, writes `nazzil_update.bat`
  (`timeout 3 → move → start → del`), launches it detached, quits.
- If running from source → opens GitHub release page in browser instead.
- Manual "Check for updates" in Settings sets `_update_check_manual = True`
  so failures show in the status bar (silent otherwise).

### Bump version manually

```cmd
python update_version.py 1.4.0
```

Rewrites `VERSION`. `config.APP_VERSION` reads it at import time;
`nazzil.spec` and `installer.iss` read it at build time.

## 12. Design Decisions Log

### Tried and abandoned

| Attempt | Why dropped |
|---|---|
| `BlurWindow` Acrylic + `WA_TranslucentBackground` | Desktop bled through if blur failed; user got jarring transparency. Replaced with solid gradient shell + `setMask`. |
| Circular 12 px traffic lights (mac-style) | Too small to click reliably on Windows. Replaced with 32×24 colored rectangles + qta icons. |
| Search button that expands on click into a QLineEdit | User said "looks half-expanded"; complexity > value. Replaced with always-visible search input between pills and sort. |
| `transparency` config flag | Pointless once acrylic was removed. Don't re-add. |
| `OPAQUE_FALLBACK_QSS` override | Was a fallback for blur-failed path. Removed when blur was. |
| Emoji glyphs (`⚙ 🔐 ☰ 🔗 🔍 ✕`) on chrome buttons | Inconsistent rendering, low quality. Replaced with `qtawesome` MDI6 vector icons. |
| Unicode `[:n] + "…"` truncation for long titles | Breaks on Arabic/emoji + clipped action buttons off-screen. Replaced with `_ElidingLabel` + `QFontMetrics.elidedText`. |
| `plyer` for desktop notifications | Replaced by `QSystemTrayIcon.showMessage` (no extra dep). |
| `pystray` for tray | Replaced by `QSystemTrayIcon` (Qt native). |
| `setLayoutDirection` on the title bar in RTL | Broke the traffic-lights pinning. Fixed by force-LTR on `WindowControls` and manual cluster reorder in `apply_layout_direction`. |

### Kept (and why)

- **Frameless window + setMask for rounded corners** — works without compositor translucency, no desktop bleed-through, no acrylic dep.
- **Custom WM_NCHITTEST** — preserves Windows Aero-snap + native edge resize without a third-party frameless library.
- **`_ControlButton = _Bullet` alias** — some legacy callsites still import `_Bullet`; the alias avoids ripple-edit risk.
- **Hidden `search_btn = search_entry` alias** — keeps any code that referenced the old icon-only button from breaking.
- **Translator's `if code == Translator.lang(): return` guard** — prevents double-loading. *Caveat:* test code must use a different starting language before flipping.
- **`size_on_disk` is the ONLY source for the completed line's size** — never fall back to `size_bytes` (single-stream from the progress hook), which caused the long-running "7 MB but file is 200 MB" bug.

### Qt QSS box-model gotcha (write this down)

`min-width: N` in QSS treats `N` as the **content** size; Qt adds the
border on top. So `min-width: 30 + border: 1px` renders at 32 px.
To get a final visible size of N, use QSS `min/max-width: N - 2*border_px`.
This bit us twice on action buttons and chrome buttons.
